## Branch naming

- `feature/<short-slug>`
- `fix/<short-slug>`
- `docs/<short-slug>`

## Lifecycle

1. Branch from `main`
2. Keep up to date with `main` via rebase
3. Open PR early for feedback if scope is large
4. Merge via squash once approved and checks pass
